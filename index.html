About This Guide

This comprehensive study guide is designed to help you transition from software engineering to network engineering, specifically tailored for joining Trischel Fabric (Pvt) Ltd's network team in Thulhiriya, Sri Lanka. Trischel recently underwent a major IT infrastructure separation from MAS Holdings, establishing their own independent and secure network infrastructure.

üöÄ Hands-On Practical Projects

Just like coding projects build your software engineering portfolio, these network projects will build your practical networking skills!

Project 1: Home Network Lab Setup

Goal: Build a virtual network infrastructure from scratch

What to do:

  • Install GNS3 or Packet Tracer
  • Create a network with 3 routers, 4 switches, and multiple VLANs
  • Configure DHCP, DNS, and routing between networks
  • Implement inter-VLAN routing
  • Document your network with diagrams

Skills gained: Basic topology design, routing, switching, VLAN configuration

Project 2: Secure Corporate Network Simulation

Goal: Design a secure multi-site enterprise network (similar to Trischel's setup)

What to do:

  • Create a headquarters and branch office topology
  • Configure site-to-site VPN between locations
  • Implement firewall rules and ACLs
  • Set up DMZ for web servers
  • Configure network segmentation (Admin, Production, Guest networks)
  • Test connectivity and security policies

Skills gained: VPN configuration, security policies, network segmentation, enterprise architecture

Project 3: Network Monitoring Dashboard

Goal: Build a network monitoring system (leverage your coding skills!)

What to do:

  • Use Python to create network monitoring scripts
  • Implement SNMP monitoring for devices
  • Build a dashboard to visualize network health (use Flask/Django)
  • Create alerts for network issues (email/SMS notifications)
  • Monitor bandwidth usage, packet loss, latency
  • Log and analyze network traffic patterns

Skills gained: Network monitoring, SNMP, Python networking libraries, automation

Project 4: Active Directory Lab Environment

Goal: Set up enterprise identity and access management (critical for Trischel)

What to do:

  • Install Windows Server in VirtualBox/VMware
  • Configure Active Directory Domain Services
  • Create OUs, users, groups, and GPOs
  • Set up DNS and DHCP integration
  • Configure file sharing with permissions
  • Implement network-level authentication
  • Practice domain joining and user management

Skills gained: Active Directory, Windows networking, authentication protocols, enterprise user management

Project 5: Network Automation with Python

Goal: Automate network configuration and management (your software engineering advantage!)

What to do:

  • Use Netmiko or Paramiko libraries to connect to network devices
  • Write scripts to backup router/switch configurations
  • Create automated VLAN provisioning scripts
  • Build a tool to audit network security settings
  • Implement configuration templates with Jinja2
  • Use Git for version control of network configs

Skills gained: Network automation, Python networking, DevOps for networks, configuration management

Project 6: Wireless Network Design

Goal: Design and secure a corporate wireless infrastructure

What to do:

  • Plan wireless coverage for office and production floor
  • Configure multiple SSIDs (Employee, Guest, IoT)
  • Implement WPA3 Enterprise with RADIUS authentication
  • Set up wireless VLANs and security policies
  • Configure QoS for VoIP and critical applications
  • Test roaming and signal strength

Skills gained: Wireless networking, RADIUS, enterprise WiFi, security best practices

Project 7: Network Troubleshooting Scenarios

Goal: Practice diagnosing and fixing network issues

What to do:

  • Use Wireshark to capture and analyze network traffic
  • Diagnose connectivity issues (ping, traceroute, nslookup)
  • Troubleshoot DHCP and DNS problems
  • Identify bottlenecks and performance issues
  • Analyze packet captures for security threats
  • Create a troubleshooting methodology document

Skills gained: Packet analysis, troubleshooting methodology, diagnostic tools, problem-solving

Project 8: IoT/Manufacturing Network Simulation

Goal: Build a network for IoT devices (relevant for fabric manufacturing)

What to do:

  • Design segregated network for production systems
  • Simulate sensors and IoT devices (use Raspberry Pi or virtual)
  • Implement OT/IT network separation
  • Configure QoS for real-time data transmission
  • Set up data collection and MQTT protocol
  • Test network resilience and redundancy

Skills gained: Industrial networking, IoT protocols, OT security, high-availability design

Project 9: Network Documentation Portfolio

Goal: Create professional network documentation (show this in interviews!)

What to do:

  • Document all your lab projects with network diagrams
  • Create IP addressing schemes and VLAN tables
  • Write standard operating procedures (SOPs)
  • Build a troubleshooting knowledge base
  • Create configuration templates and best practices
  • Use tools like Draw.io or Lucidchart for diagrams
  • Host documentation on GitHub as portfolio

Skills gained: Technical documentation, network design, professional portfolio building

Project 10: Simulated Network Migration

Goal: Practice what Trischel actually did - network separation/migration

What to do:

  • Create a "before" network topology (shared infrastructure)
  • Design an "after" topology (independent infrastructure)
  • Plan and document the migration process
  • Migrate Active Directory to new domain
  • Set up new firewall rules and security policies
  • Test connectivity and business continuity
  • Create rollback plan and disaster recovery procedures

Skills gained: Network migration, change management, business continuity, risk assessment

üí° Tips for Maximum Learning

1. Document Everything

Keep a GitHub repository with your configurations, scripts, and network diagrams. This becomes your portfolio that you can show to Trischel during interviews!

2. Break Things on Purpose

Once a project works, intentionally misconfigure it and practice troubleshooting. This builds real-world skills that you'll need on the job.

3. Combine Software + Networking

Use your Python skills to automate network tasks. This makes you stand out from traditional network engineers and gives you a unique competitive advantage!

4. Share Your Learning

Write blog posts or create YouTube videos about your projects. Teaching others reinforces your knowledge and builds your professional brand.
